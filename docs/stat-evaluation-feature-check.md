# 스탯 평가 시스템 기능 구현 현황

스펙 대비 현재 구현 여부입니다.

---

## ✅ 구현됨

| 기능 | 설명 | 구현 위치 |
|------|------|-----------|
| **스탯 정의 (기본)** | 7개 카테고리(기술, 신체, 전술, 심리, 인지, 태도, 관리) 및 카테고리별 세부 항목 | `src/lib/statDefinition.ts` (고정값) |
| **기입 평가 (선수)** | 리커트 1~5점으로 평가 | `src/app/coach/players/evaluate/page.tsx` |
| **평가자 (지도진)** | 지도 체크된 코치가 평가자로 선택 가능 | 평가 페이지에서 평가자 드롭다운 |
| **평가 대상 선수** | 평가할 선수 선택 드롭다운 | `src/app/coach/players/evaluate/page.tsx` |
| **팀 스택에서 합쳐질 코치 선택** | 팀 스탯(팀 스택) 창에서 집계에 포함할 코치 체크박스 선택, 자동 계산 | 팀 관리 팀 스택 모달, 선수 관리 팀 스택 모달 |
| **스탯 정보 (팀)** | 선수 전체 목록 + 전체 평균 + 항목별 평균 | 선수 관리 → 팀 스택 (집계) |
| **스탯 정보 (선수)** | 선수 본인만 전체 평균·카테고리별 평균 조회 | `src/app/player/stats/page.tsx` (내 스탯) |
| **스태프 평가** | 코치/프론트에 대한 스탯 평가 (기입, 리커트) | `src/app/coach/teams/evaluate/page.tsx` |
| **개인 전술 데이터** | 선수 개인 전술 데이터 → 코치 선수 개인 데이터·기록관 연동 | `src/app/player/analysis`, `src/app/player/archive` |
| **기록관 (선수)** | 선수 파트 기록관 (제출한 경기별 데이터 보기) | `src/app/player/archive/page.tsx` |
| **스탯 정의 (관리자 설정)** | 카테고리 활성화, 카테고리당 3/5/7개 항목·항목명 입력, 팀별 저장 | 코치 팀 관리 → 팀 수정 시 «스탯 정의» 블록 |
| **평가 진행률** | 설정된 평가 항목 기준 진행률(%) 표시 | 코치 팀 관리 → 팀 이름 클릭 시 펼쳐지는 명단 위 «평가 진행률» (완료 건수/최대 건수, 평균 항목 채움 %) |
| **장단점 (팀)** | 항목별 평균 상위 5위·하위 5위 표시 | 코치 팀 관리 → 스탯 보기 → 팀 스탯 페이지 «장단점» (카테고리별 상·하위 5명) |
| **장단점 (선수)** | 선수 본인 강점·약점(상/하위) 확인 | 선수 «내 스탯» 페이지 «강점·약점» (상위 3개·하위 3개) |
| **항목별 중요도** | 카테고리별 가중치(합 100%), 집계 시 가중 평균 반영 | 팀 수정 → 스탯 정의에서 카테고리별 중요도(%) 입력 |
| **평가자 선택 (설정 화면)** | 코칭 스텝 중 평가자(지도) 지정, "모두 선택"·"전체 해제" | 팀 수정 → 저장된 명단 위 «평가자 모두 선택»·«평가자 전체 해제» |
| **평가 방법 선택 (측정 평가)** | 카테고리별 기입(1~5) vs 측정(단위 지정 후 수치 입력) | 팀 수정 → 스탯 정의에서 평가 방식·단위 설정, 스태프/선수 평가 페이지에서 측정 입력, 표시 시 값+단위 |
| **기입 평가 (선수 자기평가)** | 선수가 본인 항목만 평가 (1~5 또는 측정) | 선수 대시보드·내 스탯 «자기평가» → /player/self-evaluate, evaluatorStaffId "self"로 저장, 내 스탯 집계에 반영 |

## 요약

- **들어 간 것:** 스탯 정의(고정·관리자 설정), 기입/측정 평가(카테고리별 방식·단위), 평가자/평가 대상 선수 선택, 평가자 설정(모두 선택·전체 해제), 팀 스택·코치 선택·자동 집계, 팀/선수 스탯 정보, 평가 진행률, 장단점(팀/선수), 항목별 중요도(가중 평균), 선수 자기평가, 개인 전술 데이터·기록관 연동, 선수 기록관.
- **역할 정리:** 선수는 스탯 조회 + 자기평가 가능. 코치 평가·측정 입력은 코치만.
- **미구현:** 스펙 문서 기준으로는 없음. 추가 요청 시 확장.

원하시면 위 미구현 항목 중 우선순위 정해서 설계·구현 순서 제안해 드리겠습니다.
